# Ways to Add CSS in a React Application

## 1. Normal CSS (Global CSS Files)

This is the most basic and commonly used approach.

**Example files:**

-   `index.css`
-   CSS classes written in the traditional way
-   Applied using `className` in JSX

**Example:**

```jsx
<img className="logo" />
```

```css
.logo {
    width: 100px;
}
```

**This approach is:**

-   Easy to use
-   Beginner-friendly
-   Suitable for small projects

## 2. SASS / SCSS

SASS and SCSS are **CSS preprocessors**.

**Basic idea:**

-   CSS with extra features ("superpowers")

**Notes:**

-   SASS / SCSS improve the CSS writing experience
-   Syntax is more powerful than plain CSS

**Limitations:**

-   These styles **do not scale well** for large React applications
-   Not commonly used in modern production React apps
-   Not the recommended approach for large-scale React applications today

> [SASS](https://sass-lang.com)

## 3. Styled Components

Styled Components is a **popular React-specific styling solution**.

**Key points:**

-   CSS is written inside JavaScript
-   Styles are scoped to individual components
-   Widely used in large-scale applications

**Basic idea:**

```jsx
const Button = styled.button`
    background-color: blue;
    color: white;
`;
```

**Advantages:**

-   Component-level styling
-   No global CSS conflicts
-   Clean, modular, and maintainable code

> [Styled Components](https://styled-components.com)

## 4. CSS Libraries & UI Frameworks

Another major approach is using **pre-built UI libraries**.

These libraries:

-   Provide ready-made styled components
-   Reduce the need to write custom CSS
-   Speed up development
-   Are widely used in the industry

### 4.1 Material UI (MUI)

-   A very popular React UI library.
-   Provides pre-styled and customizable components.

**Example:**

-   Instead of writing our own `<button>`
-   We import a `Button` component from Material UI
-   It already includes:

    -   Hover effects
    -   Rounded corners
    -   Colors
    -   Fonts

> [Material UI](https://mui.com/)

### 4.2 Bootstrap

-   One of the oldest and most popular UI frameworks.
-   Works with HTML, CSS, JS, and React.
-   Provides utility classes and pre-built components.
-   Uses predefined classes for styling.

> [Bootstrap](https://getbootstrap.com)

### 4.3 Chakra UI

-   Modern React component library.
-   Focuses on simplicity and accessibility.
-   Component-based styling approach.

> [Chakra UI](https://chakra-ui.com)

### 4.4 Ant Design

-   A large and feature-rich React UI library.

> [Ant Design](https://ant.design)

# Tailwind CSS

> [Tailwind CSS](https://tailwindcss.com) says:
>
> "Rapidly build modern websites without ever leaving your HTML."

**Key points:**

-   We can build and style websites without switching between HTML/JSX and CSS files.

-   Styling is done directly using utility classes.

-   Tailwind CSS is **not React-specific**.

-   It is a generic framework that works with many frontend frameworks.

## Installing Tailwind CSS

> [Install Tailwind CSS with Parcel](https://tailwindcss.com/docs/installation/framework-guides/parcel)

```bash
npm install tailwindcss @tailwindcss/postcss
```

### PostCSS

-   PostCSS is a tool for transforming CSS using JS.
-   Tailwind CSS uses PostCSS behind the scenes.

`.postcssrc` → configuration file for PostCSS

```js
{
  "plugins": {
    "@tailwindcss/postcss": {}
  }
}
```

**Explanation:**

-   This configuration tells PostCSS that Tailwind CSS is being used in the project.
-   The bundler (Parcel) uses PostCSS to process Tailwind styles.

```css
@import "tailwindcss";
```

-   Imports Tailwind CSS into the project.

## Pros and Cons of Tailwind CSS

### Pros

1. **No switching between HTML and CSS files**

-   Styling is done directly using utility classes.

2. **Faster and easier styling**

3. **Lightweight and optimized**

-   Tailwind includes only the CSS classes actually used in the project.

-   Unused classes are removed during the build process.

-   Even if a class is used multiple times, it is included only once in the final CSS bundle.

-   Results in minimal CSS output.

-   Reduces CSS redundancy in large team projects.

### Cons

1. It comes with an **initial learning curve**

2. **Reduced readability for large components**

-   A large number of utility classes can make JSX look cluttered.
-   CSS logic is mixed with JSX, which may reduce readability for some developers.
